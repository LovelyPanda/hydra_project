Detailed Description

Class for representing an image.

This data is usually created using image loader and used for creating some kinds of texture objects. It incapsulates:

  • pixels (bytes)
  • format of bytes (mode)
  • size (width and height in pixels)

The pimpl idiom is used to separate interface from implementation.

Warning:
Pixel data which you put (by constructor or setData() method) will be deleted by Image' implementation so you shouldn't delete it yourself. Data which you gain from getData() method also shouldn't be deleted, as deletion will corrupt Image object.
